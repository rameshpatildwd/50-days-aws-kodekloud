# Day 17 – Create IAM Group

## Task / Requirement
As part of access management setup in AWS, the Nautilus DevOps team needs to organize IAM users using groups.
An IAM group allows permissions to be managed collectively for multiple users.

The task is to create an IAM group with the specified name.

**Requirement details:**
- AWS Service: IAM
- Resource Type: IAM Group
- Group Name: iamgroup_mark
- Issue: IAM group does not exist
- Goal: Create the IAM group

---

## Steps Performed
- Logged into the AWS Management Console
- Navigated to **IAM → User groups**
- Clicked on **Create group**
- Entered the group name **iamgroup_mark**
- Completed the group creation process
- Verified that the IAM group was successfully created

---

## Learnings / Key Takeaways
- IAM groups simplify permission management for multiple users
- Permissions are assigned to groups instead of individual users
- Users can be added or removed from groups without changing policies
- Using groups helps maintain consistent and scalable access control
