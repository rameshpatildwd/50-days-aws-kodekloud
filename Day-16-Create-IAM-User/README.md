# Day 16 – Create IAM User

## Task / Requirement
As part of AWS account setup and access management, the Nautilus DevOps team needs to create individual user identities.
An IAM user is required to represent a person or application that needs access to AWS resources.

The task is to create a new IAM user with the specified name.

**Requirement details:**
- AWS Service: IAM
- Resource Type: IAM User
- User Name: iamuser_rose
- Issue: IAM user does not exist
- Goal: Create the IAM user

---

## Steps Performed
- Logged into the AWS Management Console
- Navigated to **IAM → Users**
- Clicked on **Add users**
- Entered the user name **iamuser_rose**
- Completed the user creation process
- Verified that the IAM user was successfully created

---

## Learnings / Key Takeaways
- IAM users represent individual identities within an AWS account
- Creating separate IAM users helps avoid sharing the root account
- IAM users can later be assigned permissions using policies or groups
- Proper IAM user management is essential for security and access control
